

| Sigla / Palavra | Nome Completo em EN | Nome Completo em PT | Conceito |
| :---: | :---: | :---: | ----- |
| ACK | Acknowledgment  | Confirmação | É um indicador de confirmação usado no estabelecimento de conexão e encerramento de sessão. |
| Bits de Controle | Control Bits | Bits de Controle | É um campo de 6 bits que inclui códigos de bits, ou sinalizadores, que indicam a finalidade e função do segmento TCP. |
| Checksum | Checksum | Checksum | É um campo de 16 bits usado para verificação de erros do cabeçalho e dos dados do segmento.  Este campo está presente no TCP e no UDP. |
| Comprimento do Cabeçalho  | Header Length | Comprimento do Cabeçalho  | É um campo conhecido como ‘offset de datas’ que indica o comprimento do cabeçalho do segmento. Este campo está presente no TCP e no UDP. |
| Controle de Fluxo | Flow Control | Controle de Fluxo | É a quantidade de dados que o destino pode receber e processar de forma confiável. |
| Datagramas | Datagram | Datagramas | Também chamados de segmentos, são pequenas partes de um pacote utilizados pelos protocolos da camada de transporte para transmitir dados entre dois hosts, sendo eles: origem e destino. |
| FIN | Finish | Fim | É um bit de controle que indica que não há mais dados do remetente, além disso este pode ser usado no encerramento de sessões. |
| Handshake Triplo | Three-way Handshake | Handshake Triplo | É um processo que serve para estabelecer uma conexão entre dois hosts. |
| ISN | Initial Sequence Number | Número de Sequência Inicial  | Representa o valor inicial dos bytes que são transmitidos ao aplicativo receptor. |
| Janelas Deslizantes | Sliding Windows | Janelas Deslizantes | É um protocolo no qual o receptor normalmente envia uma confirmação após cada dois segmentos que recebe. Isso permite que o emissor transmita continuamente segmentos, desde que o receptor reconheça os segmentos anteriores. |
| MSS | Maximum Segment Size | Tamanho Máximo do Segmento | Limita o tamanho dos pacotes, ou pequenos pedaços de dados, que trafegam em uma rede, como a internet. |
| Número de Confirmação | Acknowledgment Number | Número de Confirmação | É um campo TCP de 32 bits usado para indicar que os dados foram recebidos e o próximo byte está sendo esperado pela fonte. |
| Número de Sequência | Sequence Number | Número de Sequência | É um campo TCP usado para fins de remontagem de dados. |
| Porta de Destino | Destination Port | Porta de Destino | É um campo de 16 bits usado para identificar o aplicativo de destino por porta número. Este campo está presente no TCP e no UDP. |
| Porta de Origem  | Source Port | Porta de Origem  | É um campo de 16 bits usado para identificar o aplicativo de origem por número de porta. Este campo está presente no TCP e no UDP. |
| PSH | Push | Empurrar | É um bit de controle que indica uma função push e este é usado pelo remetente para solicitar que os dados sejam entregues à aplicação de destino o mais rápido possível. |
| RADIUS | Remote Authentication Dial-in User Service | Serviço de Usuário Discado por Autenticação Remota | É um serviço que fornece serviços de autenticação, autorização e auditoria para gerenciar o acesso do usuário. |
| Reservado | Reserved | Reservado | É um campo TCP de 6 bits que é reservado para uso futuro. |
| RST | Reset | Reiniciar | É um bit sinalizador utilizado para redefinir uma conexão quando ocorre um erro ou quando o tempo limite for excedido. |
| SACK | Selective Acknowledgment | Reconhecimento Seletivo | É um recurso TCP opcional negociado durante o handshake de três vias, este serve para reconhecer quais segmentos foram recebidos e quais não foram. |
| SEQ | Sequence Number | Número de Sequência | Identifica o primeiro byte de dados no segmento que está sendo transmitido. |
| Socket | Socket | Soquete | É a combinação de um endereço IP de origem e o número da porta de origem ou o endereço IP de destino e o número da porta de destino. É usado para identificar o servidor e o serviço que está sendo solicitado pelo cliente.  |
| SYN | Synchronization | Sincronização | É um bit de controle utilizado para sincronizar números de sequência usados no estabelecimento de conexões. |
| Tamanho da Janela | Window Size | Tamanho da Janela | É um campo TCP de 16 bits usado para indicar o número de bytes que podem ser aceitos de uma só vez. |
| URG | Urgent | Urgente | É um bit de controle que representa um campo de ponteiro urgente significativo. |
| Urgente | Urgent | Urgente | É um campo TCP utilizado para indicar se os dados contidos são urgentes, este está atrelado ao bit de controle URG. |
